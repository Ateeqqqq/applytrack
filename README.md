# ApplyTrack — Job Application Tracker

A clean, full-stack web application for tracking job applications: company, role, status, and application date, all in one dashboard.

Built as a portfolio project to demonstrate full-stack fundamentals with a Python backend, a REST API, and a vanilla JavaScript frontend.

---

## Description

ApplyTrack lets you log every job you apply to, update its status as you move through the hiring process (Applied → Interview → Offer/Rejected), and see at-a-glance statistics about your job search. It's intentionally simple: no authentication, no external services, just a straightforward CRUD app.

---

## Features

- Add, edit, and delete job applications
- Track company name, job role, posting URL, status, and date applied
- Update application status as it changes
- Filter applications by status
- Search applications by company name or job role
- Live dashboard statistics (Total, Applied, Interview, Offer, Rejected)
- Responsive, dark-themed dashboard UI

---

## Tech Stack

**Frontend**
- HTML5
- CSS3 (custom properties, no framework)
- Vanilla JavaScript (Fetch API)

**Backend**
- Python
- FastAPI
- Uvicorn (ASGI server)

**Database**
- SQLite
- SQLAlchemy (ORM)

**Communication**
- REST API over JSON

---

## Project Architecture

```
Frontend (HTML/CSS/JS)
        │
        │  Fetch API (HTTP requests, JSON)
        ▼
FastAPI Backend (main.py)
        │
        │  SQLAlchemy ORM
        ▼
SQLite Database (applytrack.db)
```

The frontend is a static site that runs independently of the backend — it just needs to know the backend's URL. The backend exposes a REST API that the frontend calls using `fetch()`. FastAPI validates incoming data with Pydantic, then uses SQLAlchemy to read/write to the SQLite database file.

## API Endpoints

| Method | Endpoint               | Description                                |
|--------|------------------------|--------------------------------------------|
| GET    | `/applications`        | Get all applications (supports `?status=` and `?search=`) |
| GET    | `/applications/{id}`   | Get a single application by id              |
| POST   | `/applications`        | Create a new application                    |
| PUT    | `/applications/{id}`   | Update an existing application              |
| DELETE | `/applications/{id}`   | Delete an application                       |
| GET    | `/stats`               | Get dashboard statistics                    |

Interactive API documentation (auto-generated by FastAPI) is available at:
**http://127.0.0.1:8000/docs**

## Local Installation & Setup (Windows + VS Code)

### 1. Open the project folder in VS Code

Open the `applytrack` folder in VS Code (File → Open Folder).

### 2. Open a terminal in VS Code

Terminal → New Terminal (this opens in the project root, `applytrack/`).

### 3. Create a virtual environment

```
python -m venv venv
```

### 4. Activate the virtual environment (Windows)

```
venv\Scripts\activate
```

You should see `(venv)` appear at the start of your terminal prompt.

### 5. Install backend dependencies

```
pip install -r backend/requirements.txt
```

### 6. Start the FastAPI backend

From the project root (`applytrack/`), run:

```
uvicorn backend.main:app --reload
```

- The API will be running at: **http://127.0.0.1:8000**
- Interactive docs: **http://127.0.0.1:8000/docs**
- The SQLite database file (`applytrack.db`) is created automatically inside `backend/` the first time the server starts.

Keep this terminal running — it's your live backend server.

### 7. Open the frontend

Open a **second** terminal (or just open the file in your browser), then:

- Simply double-click `frontend/index.html`, **or**
- Right-click `frontend/index.html` in VS Code and choose "Open with Live Server" (if you have the Live Server extension installed), **or**
- Open it directly in your browser via `File → Open File`.

The frontend will make requests to `http://127.0.0.1:8000`, so make sure the backend (step 6) is running first.
